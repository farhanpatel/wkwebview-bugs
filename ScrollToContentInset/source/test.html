window.scrollTo() - content inset bug

Scroll to the bottom and hit the "Scroll to top" button, which calls window.scrollTo(0, 0). The h1 should be at the top of the viewport, but it sits beneath the nav bar. When the content insets are adjusted automatically, scrolling from the DOM using window.scrollTo() doesn't work correctly.

This fails in tab 1 which uses automatic adjustments, and succeeds in tab 2, which has manual adjustments. It succeeds in tabs 3 and 4, which are the same as 1 and 2, but the view doesn't sit beneath the nav bar or tab bar.
